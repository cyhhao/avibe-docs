# 设置向导

运行：

```bash
vibe
```

Vibe Remote 会启动本地 Web UI 并打开设置向导。

## 你会配置什么

向导会带你完成：

- 聊天平台凭证和连接模式。
- Agent 后端选择。
- Agent CLI 路径。
- 默认工作目录。
- Vibe Remote 应该响应的 channel 或 chat。
- 所选配置的健康检查。

## 优先使用向导，而不是手改配置

Vibe Remote 会把持久化配置存到你的本地用户目录。调试时可以检查配置文件，但首次设置应该走向导，因为向导会验证凭证和平台状态。

## 平台选择

使用你本来就在工作的聊天 App：

| 平台 | 适合 |
| --- | --- |
| Slack | 团队工作流、thread、Socket Mode、成熟的交互能力 |
| Discord | 社区和个人服务器 |
| Telegram | 轻量的个人 bot 和群组 bot |
| 微信 | 中国场景下的移动优先工作流 |
| Lark / 飞书 | 中国/亚洲工作区和企业聊天 |

## 后端选择

| 后端 | 适合 |
| --- | --- |
| Claude Code | 深度推理和复杂代码库工作 |
| OpenCode | 快速本地设置和可扩展性 |
| Codex | OpenAI 编码工作流 |

你可以启用多个后端，并把不同 channel 路由到不同 Agent。
