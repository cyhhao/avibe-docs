# AI 是同事，不是工具

大多数 AI 工具仍然像终端一样工作：你输入命令，模型做事，返回结果，然后停止。

Vibe Remote 从另一个信念出发：最好的人机 AI 界面，应该更像和同事协作。

## 为什么是聊天

人类通过对话、打断、观察、跟进和 thread 来协作。群聊、私信和 channel 本来就是这些行为发生的地方。

把 AI 放进聊天里，是让机器适应人类沟通模式，而不是强迫人类生活在另一个 dashboard 里。

## 同事测试

工具等待指令。

同事会完成任务，会注意到相邻的问题，会在需要判断时询问，并在重要事情变化时回来找你。

今天的 Vibe Remote 作为编码 Agent 的远程控制器已经有用。更长期的方向是 tension-driven agency：一个能感知“现实状态”和“应有状态”之间差距，并通过明确、可审计工具采取行动的 AI 同事。

## 架构

- Inbox：用户消息、代码变更、CI 结果、系统事件。
- Processing：感知 tension、推理、排序、计划。
- Outbox：可审计动作，比如发消息、修改文件或通知人类。

聊天消息只是事件流中的一个信号，而不是整个界面。
