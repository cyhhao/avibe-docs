# WeChat and Lark

Vibe Remote also supports WeChat and Lark / Feishu so the agent can live where your actual work conversations happen.

## WeChat

WeChat is useful for mobile-first personal workflows and China-native daily communication.

Use the in-app setup wizard and follow the platform-specific prompts. After setup, send a short smoke-test message in the enabled chat.

## Lark / Feishu

Lark / Feishu is useful for workspaces that already use enterprise chat, approvals, and team collaboration in that ecosystem.

Use the setup wizard to configure credentials and select the chats or channels where Vibe Remote should respond.

## Outbound proxy

When the host running Vibe Remote cannot reach WeChat or Lark / Feishu
directly, set `proxy_url` on the matching platform config block from the
Web UI. Both HTTP/HTTPS proxies and SOCKS proxies are supported (any
`socks*://` scheme — `socks4`, `socks4a`, `socks5`, or `socks5h`).
Saving the config block only persists the change;
run `vibe restart --delay-seconds 60` afterwards so the new transport
applies to live connections.

## Shared troubleshooting

| Symptom | Check |
| --- | --- |
| Messages do not arrive | Platform credential, app installation, selected chat scope |
| Agent does not start | Backend enabled, CLI path valid, working directory exists |
| Responses appear in the wrong place | Channel or chat routing settings |
| WeChat / Lark unreachable from host | `proxy_url` on the matching config block; check firewall and TLS errors in logs |
