# Telegram

Telegram is a lightweight way to control coding agents from a phone or small group chat.

## What you get

- Send tasks to a local coding agent from Telegram.
- Use DMs or groups depending on your setup.
- Keep work local while controlling it remotely.

## Setup shape

In the setup wizard, choose Telegram and provide the bot token from BotFather. Then select the discovered chat or group where Vibe Remote should respond.

## Groups and privacy mode

If you use a Telegram group, check bot privacy behavior. Depending on your bot settings, the bot may only see commands or mentions unless privacy mode is adjusted.

## Smoke test

Send:

```text
Say hello and tell me which project directory you can see.
```

If the bot does not answer, run:

```bash
vibe doctor
```

## Outbound proxy

When the host running Vibe Remote cannot reach Telegram directly, set
`proxy_url` on the Telegram config block from the Web UI. Both HTTP/HTTPS
proxies and SOCKS proxies are supported (any `socks*://` scheme — `socks4`,
`socks4a`, `socks5`, or `socks5h`). Saving
the config block only persists the change; run `vibe restart
--delay-seconds 60` afterwards so the new transport applies to live
connections.

## Common issues

| Symptom | Check |
| --- | --- |
| Group messages ignored | Bot privacy mode and mention requirements |
| Chat not listed | Send a message to the bot or group, then refresh discovered chats |
| Agent missing | Selected backend CLI is installed and visible on PATH |
| Telegram unreachable from host | `proxy_url` on Telegram config; check firewall and TLS errors in logs |
